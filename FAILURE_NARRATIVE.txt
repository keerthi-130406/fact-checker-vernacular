During development, we faced multiple challenges. First, translation models like Helsinki-NLP/opus-mt-te-en gave 401 errors, so we switched to NLLB. Quantizing the fact-check LLM caused it to output "True" for a false claim, so we kept it at full precision and quantized only the translation model. Throughput was initially 52 claims/min; implementing batch translation boosted it to 126 claims/min. We learned that batching is critical for scale. In the future, we would use a multilingual LLM like Aya-101 to eliminate the translation step entirely and deploy on a GPU to reach thousands per minute.
